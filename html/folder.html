MERN STACK

Full Stack App para enviar y recibir información desde una base de datos desarrollada de la siguiente manera:
(bcrypt js, json webtoken, material-ui, axios)

  • Publicaciones y Paginación, estas son remitidas desde el backend con un limit de 6 publicaciones por cada pagina y un skip incial desde la ultima publicación hacia la primera.

  • Detalles de Publicación, cada publicación se conviete en un boton con el cual podemos acceder a dicha publicación con los detalles mas amplios donde podremos apreciar el mensaje, etiquetas, nombre de quien creo dicha publicación, imagen, zona de comentarios y publicaciones sugeridas, la publicación de comentarios estaran disponibles si el usuario se encuentra registrado, caso contrario solo podra apreciar comentarios sin poder publicar ni dar like.

  • Creación de nuevo cuenta, inicio de cuenta con google e inicio de cuenta con usuario ya creado, todas llevan a el panel principal de las publicaciones, en el cual se abre una nueva ventada donde el usuario actual puede crear sus publicaciones, modificarlas o borrarlas, solo las publicaciones que coincidan con el id podra modificar y borrar, al resto solo podra darles like o comentarlas.

  • Publicaciones sugeridas, estas son ancladas a las publicaciones del usuario actual por medio de las etiquetas, ya que si una o varias de ellas coinciden en el tema automaticamente la aplicación busca en todas las publicaciones y se las muestra a dicho usuario en la parte inferior de su publicación con esto podria obtener temas de su interes ya que al dar click sobre ellas, la aplicación le mostrara por medio de un enlaze dicha publicación.

Otros modulos utilizados jwt-decode, cors, moment, dotenv, react-file-base64 y body-parser puedes apreciar el código en: (GitHub) / MERN Stack

API REST

API que prestar servicio como red social estructurada de la siguiente forma:
(en el video es utilizado Insomnia para simular el cliente)

  • Crear nuevo usuario: el usuario como en toda red puede registrarse usando un username, email, password, name & photo, con todas sus respectivas verificaciones, al crearse dicho usuario en la base de datos (MongoDB) se crea un id unico para el usuario, y su contraseña queda resgistrada en dicho servidor en secreto por medio de esquema de encriptación (crypto).

  • Inicio de sesión: ya creado el usuario en la base de datos este puede iniciar su sesión por medio del username o em@il & password y automaticamente se le devolvera un token de autenticación de usuario, renovandose este cada vez que el usuario cierre su sesión y vuelva a iniciarla en su aplicación.

  • Posts: ya en su cuenta creada, el usuario puede crear, modificar o eliminar posts (ojo no puede eliminar o modificar posts que no esten asociados a su id (token)), puede de obtener todos los posts de otros usuarios o usuario en especifico, también los propios o uno en especifico, puede salvar posts de otros usuarios que le interesen y por ultimo poder dar like a los posts propios y de otros usuarios (un like por post).

  • Usuarios: al igual que los posts, el usuario puede modificar los datos de su cuenta y enviar dichos cambios a la base de datos (update user), puede obtener su información, y obtener la información de otros usuarios en especifico (ver otros perfiles), en fin la api esta lista para ser instalada en un servidor y prestar servicio como una red social.

Otros modulos utilizados cookie-parser, cors, debug, dotenv, morgan y mongoose puedes apreciar el código en: (GitHub) / Api Restful

App Módulo Nativo fs

  • Esta aplicación esta estructurada utilizando EJS (Easy Java Simulations) para procesar el código html de forma fragmentada y ordenada, cuenta con una base de datos interna (archivo json) para almacenar la información que sea digitada desde la aplicación.

Apps con Vue

Creación de tres aplicaciones con el framework de Vue.js

  • App Inicio de sesión: esta aplicación esta creada solo para experimentar el inicio y la implementación de otras librerias como bootstrap en un proyecto con el framework Vue no cuenta con mayores caracteristicas.

  • App Dinamica: aplicación creada para inicar a experimentar con el ciclo de vida de Vue (created, mounted, destroyed, etc.) dicha aplicación tiene como función que al precionar sobre cada artículo, se incremente el contador situado a el lado derecho de cada artículo y al tener una cantidad mayor que los demas este se posesione dinamicamente en la parte superior, también implemente un pequeño boton que reinicia todos los contadores a cero, puedes apreciar el código en: (GitHub) / Mi App Dinamica con Vue

  • App Carrito de Compras: Esta es una aplicación que simula consumir la información de una base de datos (api json) por medio de fetch para luego ser pintada dinamicamente en el DOM, dividiendo el código en pequeños fragmentos para poder hacer la aplicación escalable a futuro, la aplicación cuenta con una tabla oculta al iniciar la aplicación, pero visible cuando inicias a realizar compras (el carrito) esta suma la cantidad de los servicios y los monto de los mismo, también multiplica la cantidad por el monto, añadi botones para sumarle o restarle a cantidad, ademas de contar con un boton que reinicia a cero la tabla (el carrito), en la parte de las tarjetas unicamente tenemos el boton para añadir la compra de el servicio a la tabla (el carrito), la aplicación es completamente adaptable gracias a bootstrap, puedes apreciar el código en: (GitHub) / App Carrito con Vue

Apps Javascript

Aplicación de consumo de api con Ajax:

  • Es una aplicación que simula consumir la información de una base de datos (api json) para pintarla dinamicamente en el DOM con javascript ajax, también utilice jQuery ajax para consumir y pintar la información dentro del carrito, la aplicación cuenta con estructura inicial en un archivo html.

  • Añadi el cdn de bootstrap al archvio html para dar los estilos y volver adaptable la aplicación a los distintos tamaños de pantallas en los dispositivos, adicional cree un archivo css que basicamente brinda los efecto a las tarjetas y a los botones de añadir al carrito.

  • La aplicación inicia con el carrito en 0 (en rojo) y dentro una tarjeta con la leyenda de vacio, cuando inician añadir productos a el carrito la cantidad incrementa y segun el numero de productos añadidos este los cuenta y muestra la cantidad (en verde), en la pestaña desplegable del carrito se añaden una tajeta por cada compra con su respectiva imagen código, cantidad segun las veces añadido y el precio multiplicado por la cantidad, cada tarjeta añadida en el carrito cuenta con boton para eliminarlo de la tarjeta, en la parte inferior se encuentra el total de los montos, y dos botones, uno para efectuar la compra que se muestra en un modal con una tabla que solicita los datos respectivos para la compra, y el otro para reiniciar el carrito a cero, todos la información del carrito es almacenada en localStorage simulando una base de datos para la persistencia de la información, puedes apreciar el código en: (GitHub) / App Carrito Javascript

CRUD Javascript

  • Estructurada en un archivo html, y realizando todos los estilos con bootstrap, dandole dinamismo con ECMAScript 6, esta fue mi aplicación final al terminar de aprender la nueva forma de escribir código javascript y crear un CRUD por medio de modulos quedando de una forma mas ordenada todo el código y permitiendo ser reutilizable y escalable a futuro.

APIs Locales

Esta es una aplicación con la cual podemos consumir todos los recursos (APIs internas) de nuestro sistema operativo.

  • Reloj: accedo al toLocaleTimeString y al precionar el boton me muestra el reloj interno, oculto con el siguente y coleque una alarma.
  • Teclado: accedo a los shortcut de teclado para despues agregarles una funcionalidad en combinación.
  • Fecha: accedo a getTime y con un setInterval obtengo un contador en reversa, dandome detalle de los días y horas que faltan para mi cumpleaños
  • Adaptabilidad: como con una media query de css, hice que el DOM antes de los 1024px colocara video y mapa, pero más pequeña la pantalla colocara solo la referencias.
  • Testeo: pasandole al DOM datos de una url, alto y ancho para que abra una nueva ventana, esta era forma antigua de testear si las aplicaciones eran adaptables a dispositivos mas pequeño.
  • Detectar Dispositivo: con User Agent obtengo la información del dispositivo por el cual esta siendo visitada la pagina web o app.
  • Activar Camara: con getUserMedia obtengo acceso a la cama del dispositivo
  • Filtro: construyo un filtro con EventTarget.
  • Mostrar Elemento al Azar: construyo un función que toma al azar uno de los objetos del DOM y lo muestra como ganador.
  • Slider Adaptable: Forma correcta de construir un slider con javascript.
  • Video Inteligente: construyo una función que se encarge de pausar la reproducción de el video, utilizando el tamaño del scroll
  • Validación de Formulario: forma correcta de construidr un formulario.
  • Narrador de Texto: construyo un narrador de texto utilizando las distintas voces que traen los dispositivos.
Cuenta con diseño adaptable y funciones como detección de conexión red saber en que sección de la pagina me encuentro entre otras, puedes apreciar el código en: (GitHub) / APIs Internas
